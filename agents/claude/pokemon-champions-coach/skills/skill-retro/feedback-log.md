# Skill feedback log

Quick, unfiled notes about times a skill in this plugin (`refresh-references`,
`move-order-coach`, or a future one) didn't do well, jotted down in passing
during a conversation rather than filed as a GitHub issue immediately. See
`SKILL.md`'s "Logging a quick note" section for how entries get added here,
and "Step 1: Gather findings" for how they get turned into issues (and
removed from this file) during a retro.

Format per entry: `- [skill-name] one-line description of what happened (date)`
